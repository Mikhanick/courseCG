Для достижения этой цели необходимо было решить ряд ключевых задач: разработать алгоритм генерации транспортной сети с не менее чем тремя перекрёстками; создать объемные модели зданий как минимум пяти архитектурных типов; реализовать механизм воспроизводимой генерации; разработать модуль интерактивного управления виртуальной камерой; интегрировать алгоритм карты теней для расчёта освещения; и провести детальный анализ производительности решения.

**Выбор методов визуализации**

При разработке системы трехмерной визуализации я столкнулся с необходимостью выбора оптимальных методов для отображения сцены. В частности, для решения проблемы видимости поверхностей я рассмотрел несколько подходов: алгоритм художника, методы Варнока и Робертса, рей-трейсинг и Z-буфер. Результаты сравнения представлены в таблице на слайде. Как видно, метод Z-буфера показал наилучшие результаты по производительности при хорошем качестве и удобстве интеграции с системой теней, что сделало его оптимальным выбором для моей задачи.

**Работа с Z-буфером**

Z-буфер представляет собой дополнительный буфер памяти, хранящий в каждом пикселе расстояние от наблюдателя до ближайшего объекта. На левой картинке вы можете видеть визуализацию Z-буфера, где более белые участки соответствуют более близким объектам. Алгоритм работает по простому принципу: для каждого фрагмента проверяется его глубина по отношению к текущему значению в буфере, и если фрагмент ближе, он заменяет значение в буфере и цвет пикселя. 

**Реализация освещения**

Для создания реалистичного освещения я использовал диффузную составляющую освещения по закону Ламберта. В общем случае яркость в точке вычисляется как интенсивность источника света, умноженная на косинус угла между нормалью поверхности и направлением к источнику света.

**Карты теней**

Карта теней представляет собой Z-буфер, сгенерированный с точки зрения источника света. Это решение особенно эффективно, так как позволяет переиспользовать уже написанный код для работы с Z-буфером. На левом изображении видны реалистичные тени от зданий, а справа показан сам буфер теней. Для солнечного освещения я использовал ортогональную проекцию, поскольку расстояние до солнца значительно превышает размеры моделируемого города, что обеспечивает параллельные лучи света и корректное отображение теней. Важно отметить, что так как Z-буфер обновляется только при изменении параметров освещения, а при использовании карт теней для каждого фрагмента проверяется только глубина этой точки в пространстве буфера карты теней, то это делает его весьма эффективным для вычислений.

**Алгоритм генерации городской среды**

Процесс создания городской среды начинается с рекурсивного разделения начального квадрата границ города на прямоугольные кварталы. Разделение продолжается до тех пор, пока длина наибольшей стороны квартала не станет менее 600 условных единиц, эквивалентных метру в реальном мире. Затем в каждом квартале создается сеть внутриквартальных дорог, после чего вдоль этих дорог автоматически размещаются здания с учетом доступного пространства. Для обеспечения воспроизводимости результатов используется фиксированное начальное значение генератора псевдослучайных чисел.

**Технологический стек**

Программная реализация выполнена на языке C++ стандарта C++20 с использованием фреймворка Qt 6.5 для создания графического интерфейса. Для распараллеливания вычислений при растеризации применена библиотека OpenMP. Разработка велась в среде Visual Studio Code.

**UML-диаграмма классов домена генерации города**

Здесь представлена диаграмма классов, в основе создания города лежит набор дорог, коллекционирующих в себе графические объекты, а также стратегия расположения дорог и стратегия выбора нужной графической модели.

На следующих трех слайдах представлены части диаграммы классов домена отрисовки. На первой части показана иерархия классов различных буферов экрана. 
На второй части диаграммы показаны такие объекты как сцена, освещение, камера и графический объект.
На заключительной диаграмме показаны команды растеризации для одновременной работы с несколькими буферами. Быстродействие этой части достигается при помощи статического полиморфизма.

**Интерфейс и управление**

Интерфейс программы предоставляет настройки для управления генерацией и визуализацией. Пользователь может настраивать освещение, параметры генерации города, качество рендеринга и угол обзора. Меню может быть представлено как в виде вкладки в основном окне, так и в отдельном плавающем окне. Управление камерой осуществляется следующим образом: перемещение с помощью клавиш W, A, S, D с возможностью ускорения клавишей Shift, а вращение — движением мыши с зажатой левой кнопкой.
На данном слайде вы можете видеть различные варианты настроек освещения, которые пользователь может изменять в реальном времени.

**Результаты исследований**

Было проведено исследование зависимости времени отрисовки квадратного изображения от размера стороны данного изображения. Была выявлена зависимость, аппроксимируемая квадратичной функцией, при чем при большем размере города также была выявлена задержка отрисовки меньшего порядка, связанная с необходимостью перебора большого количества полигонов. Замеры производились на процессоре AMD Ryzen 7 8750H, на ноутбуке был включен режим высокой производительности, сам ноутбук был подключен к сети. Данные усреднены за 50 кадров при фиксированном положении камеры.
