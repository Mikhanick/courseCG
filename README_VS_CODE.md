# Qt-проект в VS Code

## Необходимые расширения
- C/C++ (Microsoft)
- Qt Tools (Thierry Boudier)
- CMake Tools (Microsoft)
- CMake (Microsoft)

## Быстрая сборка
- `Ctrl+Shift+P` → "Tasks: Run Task" → "build"
- Или используйте `Ctrl+Shift+B` для запуска задачи сборки по умолчанию

## Отладка
- Нажмите `F5` для запуска отладки
- Перед этим убедитесь, что проект собран (см. "Быстрая сборка")
- Используйте `F9` для установки точек останова

## Qt-специфичные функции
- Для работы с .ui файлами используйте Qt Designer
- Компиляция .ui файлов происходит автоматически через uic
- Для пересборки moc файлов выполните qmake: `qmake -makefile program.pro`

## Горячие клавиши
- `Ctrl+Shift+P` - вызов палитры команд
- `Ctrl+Shift+B` - сборка проекта
- `F5` - запуск отладки
- `F9` - установка точки останова
- `F10` - шаг с обходом
- `F11` - шаг с заходом

## Проблемы и решения
1. Если IntelliSense не работает:
   - Проверьте правильность путей в `.vscode/c_cpp_properties.json`
   - Перезапустите C/C++ IntelliSense: `Ctrl+Shift+P` → "C/C++: Restart IntelliSense Engine"

2. Если сборка не удается:
   - Убедитесь, что qmake и make установлены
   - Проверьте, что Qt библиотеки установлены и доступны

3. Для обновления конфигурации:
   - Пересоздайте Makefile: `qmake -makefile program.pro`
   - Перезапустите VS Code

## Структура проекта
- `MainWindow.*` - основное окно приложения
- `renderer/` - модули рендеринга
- `.pro` файл - конфигурация сборки Qt
- `.vscode/` - настройки VS Code для проекта

## Qt-специфичные сниппеты
- `qclass` - создает шаблон Qt-класса с Q_OBJECT
- `qsignal` - добавляет объявление сигнала
- `qconnect` - создает соединение между сигналом и слотом
- `qpaint` - создает обработчик paintEvent
- `qkeyevent` - создает обработчик keyPressEvent
