# Qt Coding Guidelines для проекта

## Qt-Specific Features

### Signals and Slots
- Используйте макрос `Q_OBJECT` в классах, где планируется использовать сигналы/слоты
- Подключайте сигналы к слотам через `connect()` с использованием `&Class::signal` и `&Class::slot`
- Используйте лямбда-функции для простых обработчиков событий

### Memory Management
- Используйте родительско-дочерную иерархию Qt для автоматического управления памятью
- Для объектов, не входящих в иерархию Qt, используйте RAII и умные указатели
- Избегайте утечек памяти при использовании `new` без родителя

### Qt Containers vs STL
- Используйте `QVector<T>`, `QList<T>`, `QString` и другие Qt контейнеры
- Qt контейнеры оптимизированы под копирование и имеют copy-on-write
- STL контейнеры можно использовать, но с осторожностью

## Qt и OpenGL/Rendering
- Для рендеринга в QWidget используйте QPainter или QOpenGLWidget
- Для программного рендеринга (как в вашем проекте) используйте QImage
- QImage можно отображать через QLabel или напрямую в paintEvent

## Qt Threading
- Не обновляйте UI из потоков, отличных от GUI-потока
- Используйте сигналы/слоты для передачи данных между потоками
- Для длительных вычислений (рендеринг) используйте QThread или QtConcurrent

## UI Design
- Используйте Qt Designer для создания .ui файлов
- .ui файлы компилируются в заголовочные файлы (ui_mainwindow.h)
- Для динамического UI используйте layout'ы (QVBoxLayout, QHBoxLayout и т.д.)

## Project Structure
- Разделяйте логику UI и рендеринга (как в вашем проекте с MainWindow и Renderer)
- Используйте MVC/MVP паттерны при необходимости
- Для сложных сцен используйте QGraphicsScene/View

## QMake Specifics
- В .pro файле указывайте все исходники и заголовки
- Используйте CONFIG += c++20 для современного C++
- Добавляйте необходимые модули Qt в QT += ...
- Для OpenMP используйте QMAKE_CXXFLAGS и QMAKE_LFLAGS как в вашем проекте

## Debugging Tips
- Используйте qDebug() для отладки (как printf, но с Qt типами)
- Qt Creator имеет отличный отладчик, но VS Code с C++ extension также подходит
- Для профилирования используйте QElapsedTimer

## Performance
- Избегайте частых перерисовок - используйте update() вместо repaint() когда возможно
- Оптимизируйте рендеринг - не перерисовывайте то, что не изменилось
- Используйте QCache или другие кеширующие механизмы для тяжелых вычислений
